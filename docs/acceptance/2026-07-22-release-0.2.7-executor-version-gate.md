## 实机发现

- 重载后的持久化配置和窗口会话配置均为 v2，只有
  `/home/unitree/mimiclite-sim2real` 一个 `remote/primary` 根。
- Bridge 通过现有 VS Code Remote transport 进入 `ready`，没有发起第二条 SSH 认证。
- 远端 `zkbot.codex-remote-bridge-executor-0.2.6/dist/extension.cjs` SHA-256 为
  `596CD5985A18B5CDDC0A1A247DEF196A8D03EC80F82AE0398CF3D16A388C60F6`，与候选源码构建
  `4745AFAD844900F9389246BCEDE36D0553BA3823BAF8D85B04DA52D3BC039D7A`
  不一致。
- 根因是旧 ping 只返回协议 v4；Controller 将同协议的旧 Executor 误判为当前构建，
  因而没有执行内嵌 VSIX 升级。
- Executor `0.2.7` 安装并激活后，远端 `extension.cjs` SHA-256 为
  `4745AFAD844900F9389246BCEDE36D0553BA3823BAF8D85B04DA52D3BC039D7A`，与本地候选
  完全一致。Bridge 再次进入 `ready`，只读 `probe` 返回 `g1_1` 和
  `/home/unitree/mimiclite-sim2real`，命令实际工作目录也为该远端主根。

## 遗留风险与结论

- Controller 检测到版本不匹配后安装 Executor 并自动重载；同一候选五分钟内最多安装
  和重载两次，仍不兼容时失败关闭。
- 本次不改变 Executor 协议 v4；版本不匹配现在也会触发失败关闭和升级。
- Windows x64、OpenSSH 回退、设置恢复和完整生命周期仍待补测。
- 最终结论：**Linux x64 的 Executor `0.2.7` 安装、激活、摘要一致性和远端主根探针
  通过；自动重载触发与完整远端/MCP 固定探针仍待下一次升级链路补测。**
- 支持声明：本记录只声明 Linux x64 Controller 到 `g1_1` 的 Executor `0.2.7` 升级和
  只读主根探针，不外推 Windows、OpenSSH 回退或完整生命周期。
