# 0.2.7 Core 本地拒绝候选验收

日期：2026-07-22

## 范围

本记录只验收阶段 2C 的已实施部分：

- Remote Bridge app-server 注入并默认选择专用 named permission profile。
- 线程新建、恢复、设置更新、fork 和 turn 不能通过请求参数放宽本地策略。
- 已知本地 Shell、文件、命令、进程、模糊搜索和后台终端客户端请求在 Shim 前失败关闭。
- Core 发起的五类本地命令、文件和权限审批在到达官方 UI 前失败关闭。
- 原有远程主根和活动 VS Code Remote transport 不回归。

本记录不把官方扩展版本固定为启动门槛。实测来源扩展为
`openai.chatgpt@26.715.61943`，其内置 Codex 为 `0.145.0-alpha.27`。

## 能力边界

- 生成的 `ClientRequest` 中共有 25 个落入当前高风险本地命名空间的方法，已全部进入
  受控阻断集合。协议升级新增同类方法时，兼容测试会失败。
- 官方 app-server 接受 `codex-remote-bridge` profile，并在 `thread/start` 响应中
  返回该 active profile、`approvalPolicy=never`、只读 sandbox 和禁用网络。
- session flag 注入的 `PreToolUse` hook 为 `untrusted`。本候选不修改用户 hook 信任，
  也不启用会连带信任其他 hook 的全局绕过参数。
- 官方文档不把 hook 定义为所有专用工具的完整强制边界，因此本次结果只能标为
  `部分实施/待补测`。
- 当前 Linux 环境的 Core 文件诱饵探针在路径检查前命中 `bwrap` 环回网络初始化错误，
  不能据此证明本地文件拒绝成功。
