## 遗留风险与结论

- 待用户重载当前 Remote SSH 窗口并使用官方扩展新建任务。
- 待确认新任务到达 `0.145.0` Shim，且固定项目操作全部记录为远端操作。
- 待重跑 stdio MCP、本地窗口透传、审批、恢复和生命周期链路。
- 待在 Windows x64 原生构建 SEA Shim、收集双平台产物并分别完成实机验收。
- 最终结论：**候选自动化通过，但发布硬门禁未通过，不声明支持 `0.145.0`。**
- 支持声明：上一份完整实机基线仍为 `docs/acceptance/2026-07-18-release-0.2.7.md`。
